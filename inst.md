Seowl 마스터 플랜을 바탕으로, 대표님(또는 프로젝트 총괄)께서 각 분야별 담당자(혹은 AI 에이전트)에게 즉시 업무를 지시할 수 있도록 분야별 액션 플랜과 AI용 복붙 프롬프트를 정리해 드립니다.

이대로 각 역할의 AI에게 복사해서 붙여넣기만 하시면 바로 실무 작업이 시작됩니다.

📋 1. 서비스 기획 (Product Management)

[미션: 군더더기 없는 MVP 화면 설계와 정책 수립]
가장 먼저 '무엇을 만들고 무엇을 뺄지'를 결정해야 합니다. 초기 유저를 설득할 화면 플로우(User Journey)와 커뮤니티 규칙을 세웁니다.

🤖 기획 에이전트용 지시문 (복붙하세요):
"Activate Product Manager mode.
최상위 폴더의 seowl_master_plan.md를 읽어줘. 우리는 MVP(Phase 1)를 준비 중이야.

유저가 앱을 처음 다운로드하고 'Drive / Active / Work / Safe' 중 자신의 올빼미 유형을 선택하는 온보딩(Onboarding) 3단계의 화면 설계서(Wireframe 구조)를 텍스트로 작성해 줘.

밤의 특성상 악성 유저가 생길 수 있으니, '심야 익명 커뮤니티'에서 지켜야 할 Seowl만의 감성적이고 단호한 커뮤니티 가이드라인(정책) 5가지를 작성해 줘."

🎨 2. UI/UX 디자인 (Design)

[미션: 네이버지도와 완벽히 차별화되는 '밤의 감성' 시각화]
단순한 지도를 넘어 '힙한 야간 매거진' 느낌을 주어야 하며, 밤에 눈이 부시지 않아야 합니다.

🤖 UI/UX 에이전트용 지시문 (복붙하세요):
"Activate UI Designer mode.
seowl_master_plan.md의 '뱀파이어 모드' 컨셉을 참고해 줘.

심야 지도 화면에 쓰일 메인 컬러(Primary, Secondary, Background, Text)를 HEX 코드와 함께 제안하고, 왜 이 색이 올빼미족의 눈 피로를 줄이면서도 트렌디한지 설명해 줘.

유저가 특정 매장에 '지금 영업 중(Proof of life)'이라고 체크인했을 때, 지도상의 핀(Pin) 아이콘이 어떻게 변해야 시각적 쾌감(게이미피케이션)을 줄 수 있을지 애니메이션 효과를 기획해 줘."

📱 3. 프론트엔드 개발 (Frontend Engineering)

[미션: 매끄러운 지도 인터랙션과 토글 기능 구현]
앱의 첫인상을 결정하는 모바일 UI 구현입니다. React Native(또는 Flutter) 기반으로 프론트 단을 짜야 합니다.

🤖 프론트엔드 에이전트용 지시문 (복붙하세요):
"Activate Frontend Developer mode.
seowl_master_plan.md를 읽고 프로젝트의 톤앤매너를 파악해 줘.

React Native를 사용하여 메인 화면 코드를 작성해 줘. 상단에는 '운전자 모드'와 '뚜벅이 모드'를 스와이프해서 전환할 수 있는 애니메이션 토글(Toggle) 컴포넌트가 들어가야 해.

하단에는 '안심 지수'와 '콘센트 유무'를 껐다 켤 수 있는 필터 칩(Filter Chip) 영역 코드를 작성해 줘."

⚙️ 4. 백엔드 및 인프라 (Backend & Infra Engineering)

[미션: 심야 트래픽 방어와 실시간 위치 데이터 처리]
밤 12시부터 새벽 4시 사이에 모든 트래픽과 데이터 쓰기(생존 신고)가 폭주합니다. DB를 단단하게 만들어야 합니다.

🤖 백엔드 에이전트용 지시문 (복붙하세요):
"Activate Backend Architect mode.
seowl_master_plan.md의 '실시간 생존 신고'와 '자정~새벽 4시 트래픽 폭주' 문제에 집중해 줘.

매장의 영업 상태(True/False)와 혼잡도 데이터를 RDBMS(PostgreSQL)가 아닌 Redis에 실시간 캐싱하여 업데이트하는 Node.js 백엔드 로직 코드를 작성해 줘.

유저의 내 위치 기준 '반경 2km 이내 매장만' 빠르게 검색해서 가져오는 PostGIS(공간 DB) 쿼리문을 작성해 줘."

🚀 5. 마케팅 및 그로스 (Marketing & Growth)

[미션: 적은 돈으로 초기 1,000명의 열성 유저 획득]
콜드스타트(초기에 유저가 아무도 없는 현상)를 깨부술 날카로운 카피와 숏폼 대본을 짜야 합니다.

🤖 마케터 에이전트용 지시문 (복붙하세요):
"Activate Growth Hacker mode.
seowl_master_plan.md의 6번 '게릴라 마케팅' 파트를 숙지해.

우리는 '네이버지도 영업 중 마크 믿고 새벽에 나갔다가 헛걸음한 경험'을 타겟으로 틱톡/릴스 숏폼 영상을 하나 찍을 거야. 촬영이 쉽게 15초 분량의 유머러스하고 빡친(?) B급 감성 대본과 화면 연출 지시문을 써줘.

홍대입구 전봇대에 붙일 야광 포스터에 들어갈 가장 후킹한 1줄짜리 헤드카피를 5개 뽑아줘."

🕵️ 6. 데이터 리서치 및 오퍼레이션 (Data & Operations)

[미션: 빈 수레가 요란하지 않도록, MVP용 진짜 데이터 구축]
런칭 첫날 앱을 켰을 때 최소한 한 구역은 핀(데이터)이 꽉 차 있어야 합니다. 이를 자동화해야 합니다.

🤖 리서처 에이전트용 (Hermes-agent 등) 지시문:
"Activate Data Researcher mode.
도구(Web Search Tool, File Writer)를 사용해서 아래 임무를 수행해 줘.
타겟 지역은 오직 '서울 홍대입구역 반경 1.5km 이내'야.
현재 인터넷(지도, 블로그 리뷰 등)을 검색해서 해당 지역 내에 '24시간 운영'이라고 표기된 카페나 패스트푸드점 30곳의 데이터를 모아줘.
그리고 결과를[상호명, 주소, 콘센트 존재 여부(리뷰 바탕 추측), 심야 시간 무인 전환 여부] 속성이 포함된 hongdae_night_data.json 파일로 만들어서 로컬에 저장해."

💡 대표(Manager)의 역할

대표님은 이 코드와 기획안을 통합하고 결정만 내리시면 됩니다.
agency-agents나 hermes-agent 환경을 터미널(또는 커서 에디터)에 띄우고, 각 에이전트에게 위 1~6번 프롬프트를 차례대로 복사+붙여넣기 하십시오. 기획자 AI가 짜준 화면 설계서를 개발 AI에게 넘겨주고 코드로 바꾸라고 지시하는 식으로 '결과물을 넘겨주며 릴레이 시키는 것'이 핵심 워크플로우입니다!
